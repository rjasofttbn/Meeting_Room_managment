What is MailSend?

MailSend is a mail transport class. It is authored by Andy Prevost, one of the original authors of PHPMailer and PHPMailer2.

MailSend is a full featured class that includes support for SMTP, Sendmail and PHP Mail(). It is under 40Kb and is safe and secure.

Order of processing:

MailSend requires minimal setup on your part. Essentially, you suppoly the Sender, Recipient, Subject, and your HTML message. Send, and:

  • MailSend will get the SMTP server for your domain
  • MailSend will attempt to use SMTP with no authentication
    - should not be an issue on your own domain
  • If SMTP fails, MailSend will determine if Sendmail is installed and use Sendmail.
  • If Sendmail fails, MailSend will default to PHP Mail(). Note that this is rare, and likely only happen on Windows servers that do not have a sendmail wrapper installed (most do).

This specifically means that MailSend offers the greatest range of MTA's. Your own SMTP server, Sendmail, PostFix, Qmail, Exim, and many many more. Virtually all MTA's provide a sendmail wrapper.

How is MailSend different from PHPMailer2?

While I was admin and author of PHPMailer, many users requested a slimmed down version of PHPMailer. I had launched PHPMailer-Lite that was a slimmed down version of PHPMailer - it did not support SMTP, however. MailSend is designed specifically for light use, small mailing lists, and form submission purposes.
MailSend is planned to be a part of the mail and form system. PHPMailer2 is slated to support a wide array of plugins that will be unique to PHPMailer2.

Basic Usage

Download the MailSend zip file. Unzip it, and put the MailSend.php on your server. In the example below, we assume the MailSend.php file is in the root, along with the test.php script. Your directory strure should look like: 

/MailSend.php
/test.php

There is NOTHING else needed. This basic setup and test will use the form, MailSend.php, and send the email using PHP's built in mail() wrapper.

You will need to edit the test.php script to include your own personalized information.
There is nothing to edit or change in MailSend.php. Every aspect can be controlled from the form. Here is what the form code looks like. And, there is only ONE part to edit: change the SetRecipient to be your own email address (in the example below, look for the red line). Once you do that, you can test the form and it will send you an email with the form data you fill in during the test. One quick note: the example below includes two honey pot fields to catch potential hackers. The code is there to show how that works.
PS. This is EXACTLY the same test script included in the package: 

<?php
require_once 'MailSend.php';

$mail = new codeworxtech\MailSend;

$mail->SMTP_Debug = 0; // 0 = off, 1 = basic, 2 = advanced (server)

$mail->SetSender( ['yourname@yourdomain.com'=>'Full Name'] );
$mail->SetConfirmRead('confirmname@yourdomain.com');

$mail->AddRecipient( ['recipient@domain.com'=>'Recipient Name'] );
//$mail->AddCC( ['cc@domain.com'=>'CC1 Name','cc2@domain.com'=>'CC2 Name'] );
//$mail->AddBCC( ['bcc@domain.com'=>'BCC1 Name','bcc2@domain.com'=>'BCC2 Name'] );

$mail->SetSubject('Example mail');
$mail->MessageText = 'Example plain-content!';
$mail->MessageHTML = 'embedded.html';

$mail->AddAttachment('screen_shot_sample_form.gif');
